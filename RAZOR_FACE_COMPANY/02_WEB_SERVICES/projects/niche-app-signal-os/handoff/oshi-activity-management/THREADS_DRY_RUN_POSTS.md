# Threads DRY_RUN Posts - Oshi Activity Management

Date: 2026-06-06

Status: DRY_RUN only. Do not post.

## Variant A - 現場ごとに散らかる情報

ライブの日程はカレンダー。  
当落や入金はメールやスクショ。  
グッズは写真。  
遠征費は決済履歴。  
感想はメモ。  

でもあとで見返したい時は、だいたい「この現場どうだったっけ？」なんですよね。  

現場ごとに、参戦ログ・グッズ・遠征費・思い出メモをまとめる小さなボードがあったら便利だろうか。  
ログインなしの試作版で、まず需要だけ見たい。

Expected signal:

- Comments about scattered current workflows.
- Requests for fields like ticket_status, companion, emotion_tag.
- Saves from people who attend multiple events.

## Variant B - グッズ/遠征費でお金が見えない

推し活で地味に見えなくなるの、チケット代よりも周辺費かもしれない。  

交通費、ホテル、食事、グッズ、交換用の追加購入。  
全部楽しい支出だけど、現場ごとに見返すと意外とまとまっていない。  

ライブ参戦ログと一緒に、グッズと遠征費をメモできる小さなローカル管理ボードを試作中。  
家計簿ほど重くなく、現場ごとに残せるくらいがちょうどいいのか知りたい。

Expected signal:

- Comments about budgeting.
- Mentions of travel-heavy fans.
- Requests for monthly/yearly totals.

## Variant C - 思い出が残らない

ライブ直後は全部覚えている気がする。  
でも数か月後に見返すと、席、セトリ、MC、買ったグッズ、同行者との話が少しずつ薄れている。  

写真だけだと残らない情報がある。  
メモだけだと現場とつながらない。  

参戦ログ、グッズ、遠征費、思い出メモを現場ごとに残す試作版を作った。  
推し活の記録って、どこまで残せると嬉しいんだろう。

Expected signal:

- Comments about memory, diary, and emotion tags.
- Requests for photos or highlights.
- Pushback if management feels too utilitarian.

## Comparison Rule

Use A as the primary hypothesis. Use B and C to compare whether the audience responds more to money management or memory preservation.

Do not publish these automatically. Owner/GPT review is required before any live use.
